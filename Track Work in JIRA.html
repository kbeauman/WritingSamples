Epics, Stories, and Sub-tasks

MBP Documentation uses epics to track a single release deliverable involving one or more components. For example, an upcoming release, vX.Y.Z, may consist of enhancements for Retail Deposits, Commercial Deposits, CAPE RAS, and Core Fusion. Each of these components must have a Release Notes document. The epic for this scenario is MBP vX.Y.Z GA Release Notes. Each component has a story for the Release Notes, e.g., MBP vX.Y.Z Commercial Deposits Release Notes, and each sprint has a sub-task, e.g., MBP vX.Y.Z Sprint 5 Commercial Deposits Release Notes. Each story is linked to the parent epic and each sub-task to the parent story.

Create a release project

  1. On the Jira top navigation, select Create. The Create issue dialog appears.
  2. Complete the required fields:
    1. For Project, select MBP Documentation.
    2. For Issue Type, select Epic.
    3. For Epic Name, use the naming convention MBP v#.#.# GA Deliverable, where # is the release version and Deliverable is the document type, e.g., Release Notes, Product Overview, or Enhancement Guide.
    4. Complete the remaining required fields with the appropriate release documentation information.
  3. Select Create. The dialog closes and a message appears with the epic's issue key. Optionally, you can select the issue key to open the epic.
  4. Repeat the process to add a story:
    1. For Issue Type, select Story.
    2. For Story Name, use the naming convention MBP v#.#.# Component Deliverable, where Component is the Modern Banking Platform component.
    3. For Epic Link, select the new epic you just created previously.
  5. Select Create. The dialog closes and a message appears with the story's issue key.
  6. Select the issue key to open the story.
  7. On the More menu, select Create sub-task and use the naming convention MBP v#.#.# Sprint # Component Deliverable, where Sprint # is the development sprint being documented.

Using this procedure ensures all related issues are linked together and organized in a standard structure.

Develop Release Notes

The process of developing Release Notes can be applied to Enhancement Guides and Product Overviews. However, for the sake simplicity, only Release Notes is discussed. Also, this article does not replace or supersede Release Notes Best Practices, but provides an alternative method for developing this publication.

  1. Determine the Release Version
    1. On the Jira top navigation, open the Projects menu and select the project to be documented. A page appears with the project left navigation.
    2. On the left navigation, select Releases. A list of Releases appears.
    3. Select the release Version to be documented. A summary page appears.
  2. On the title bar, select Release Notes. A list appears with all issues for the release and each issue key is linked to the issue.

This list is also available in HTML syntax at the bottom of the page. You can copy the syntax, paste into Notepad, and save as a HTML file, which can then be opened in Word to give you a quick outline.

NOTE: This report is a snapshot of the release with issue keys and issue names. Any changes to the release, such as issues being descoped or added, require manual revisions.

Track Work in Jira

Jira offers several options you can use to follow specific tickets, or issues. Building a dashboard can help you easily track issues as they move through a workflow or move from one release to another.

Dashboard

The primary tool for frequent viewing issues of interest is the dashboard. A dashboard consists of one or more panes called Gadgets that display lists, graphs, and other helpful aids. To get started, build a dashboard with a gadget listing all issues assigned to you. Then, add a gadget to list all issues you have selected to watch.

To build a dashboard

  1. In Jira, go to Dashboards > Manage Dashboards. The Manage Dashboard page appears.
  2. Select Create new dashboard. The Create New Dashboard page appears.
  3. Complete the form:
    1. For Name, enter a descriptive label.
    2. On the Start From menu, select My Issues.
  4. Select Add. The Manage Dashboard page appears with the My Issues dashboard listed.

The My Issues dashboard contains a gadget listing all Jira issues assigned to you. You can access this dashboard from the Dashboards menu.

Add a Gadget

Jira offers a Watch feature that allows you to select specific issues and receive updates on any changes. However, these email notifications can become overwhelming when watching several issues. Instead, you can add a gadget to the My Issues dashboard that lists you’re watching issues.

To add a gadget

  1. In Jira, go to Dashboards > My Issues. The My Issues dashboard appears.
  2. Select Add gadget. The Add a gadget dialog appears.
  3. Select Load all gadgets.
  4. Scroll to Watched Issues and select Add gadget. The Watched Issues gadget is added to the dashboard in Edit mode.
  5. Close the Add a gadget
  6. Optionally, you can change the number of issues listed per page, as well as delete, add, and rearrange the elements as desired. It is recommended to select Updated from Add fields… This allows you to identify any issues updated since your last session.
  7. Select Save. The list of watched issues appears.

Add a Filter

When you need to view a list of issues with one or more common criteria, you can build a filter and associate the filter to the Filter Results gadget. Consider the following resources to aid in this procedure:

  • Advanced searching – operators reference
  • Advanced searching – fields reference

The following procedure results in a list of Story issues assigned to a specific release.

To add a filter

  1. In Jira, select in the Search text box. A list of issues and projects appears.
  2. Select View all issues. The advanced search page appears.
  3. In the Search text box, enter the field label to search by, such as fixVersion to view all issues assigned to a specific release. As you type a list of options appears containing the characters you have entered.
  4. Select the desired label. A list of operators appears.
  5. Select the desired operator and enter the keyword to search by.
  6. You can continue adding filter criteria, such as issueType = Story.
  7. With all filter criteria entered, select Search. A list of issues matching your filter criteria appears.
  8. Select Save as. The Save filter dialog appears.
  9. Enter a descriptive name for the filter and select Save. The dialog closes and the name appears above the Search text box.

You can now assign this filter to the Filter Results gadget to view a list of these issues on your dashboard.

Kanban Board

Use the MBP Documentation Kanban board to quickly view project issues. The Kanban board is available on the left navigation any time you select a project from the Project menu. See What is kanban for additional information.

MBP Documentation Kanban board consists of columns representing each workflow status and a count of issues in the status. Each swimlane represents an epic and contains associated story and sub-tasks. You can expand or collapse a swimlane for easier viewing of a specific epic. You can drag issue tiles from one status column to another within compliance of the workflow to quickly update the status of the issue.

The filter menu includes filters to limit the view to MBP Documentation project issues only, and further limit to a specific release. Note that you must remove the filter for one release before applying another release filter. Otherwise, you are filtering for issues assigned to both releases which most likely results in no issues.
