Manage Help Files

Version Control and file backup.

E7 uses TortoiseSVN version control for managing files. TortoiseSVN Client is a free open-source program for version control and is available at http://tortoisesvn.net/.

Help File Folder Structure

E7 employs the industry standard of using trunks, branches, and tags to manage development cycles. The current policy for Help files is to reserve trunk folders as the master from which all release versions are branched from. Therefore, for any given release version you must cut a branch folder:
  • branch
  • '--5.3.1
  • '--5.4.0
  • trunk
  • tag
  • '--5.0.0
  • '--5.1.0
  • '--5.2.0
  • '--5.3.0
Tag folders are used to store released versions for reference and possible roll-backs.
It is recommended to Check files out to a branch folder on your hard drive. This is referred to as the Working Copy. Using the previous example, your Working Copy consists of a folder named branch, which contains folders 5.3.1 and 5.4.0.
CAUTION:
Each folder level of the Working Copy contains a hidden folder "\.svn". This is used by TortoiseSVN for source control and should be considered off limits by novice users.

Following a release, the branch version is used to generate a tag version and then merged with the trunk. See Merge a Branch to the Trunk for instructions on the post-release process.
