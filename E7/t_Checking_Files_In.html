Checking Files In

Backing up your revisions.

At regular intervals, typically end-of-day, check-in your changes to the network. This servers two purposes, to provide a secure backup of your files and to allow colleagues access to your revisions. When multiple users are working from the same directory, each check-in procedure must begin with an update of your SVN log (a hidden folder residing in each of your Working Copy folders. However, when you are the sole author of these files you may begin this process at Step 4:

  1. In Windows Explorer, right-click the folder that contains all the files you changed and/or added to the help and select SVN Update from the menu. A TortoiseSVN update window displays files of the repository which have been added or updated by others along with the update action in the Action column. See Update Your Working Copy With Changes From Others in the TortoiseSVN Help for additional information.
  2. Click OK to close the TortoiseSVN update window.
  3. Resolve any Conflicted files. See Resolving Conflicts in the TortoiseSVN Help for instructions.
  4. In Windows Explorer, right-click the folder that contains all the files you changed and/or added and select SVN Commit from the context menu. After scanning your PC and the network, a TortoiseSVN Commit dialog box displays a list of all added and revised files in your Working Copy, along with a checkbox next to the name. TortoiseSVN automatically marks this box for files that you checked out and modified. Unmodified and new files have a clear checkbox.
  5. Check the box for the new files you want to upload. Alternatively, check Select/deselect all to upload all changes.
  6. Click OK. The TortoiseSVN Commit progress window displays each file as it is written to the repository. OK is enabled when the process completes.
  7. Click OK. The TortoiseSVN Commit progress window closes.
Ideally, the green checkmark icon () displays. However, in the event the icon stays red (), it is recommended to open each subfolder and verify all new files have been added. Occasionally, XMetaL support files cause the icon to remain red. If you attempt a second commit and no files display in the dialog box, open the Repo-browser from the context menu and verify your revisions now reside on the repository server.
