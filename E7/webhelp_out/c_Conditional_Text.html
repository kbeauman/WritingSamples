Conditional Text

Set this attribute to use a common doc-set for multiple versions.

There is an entire section on this topic in XMetaL Help>Working with DITA>Conditional Text. Also, a brief, but informative blog at http://flowwrite.blogspot.com/2008/03/using-dita-for-conditional-text.html. Here are a few additional notes that may not be covered, or are specific to our application of this feature.

Here are some best practice tips from the fine folks at XMetaL:
  • To make a whole topic conditional, conditionalize the corresponding topic reference in the DITA map file. See the following section on Entire Topics for additional information.
  • Do not conditionalize content that is "common to all."
  • If your content will be translated, it is recommended to conditionalize entire sentences, not parts of sentences.

At the moment, Conditional Text is only used in SONUS for Fiat and Chrysler only content. There are also one or two instances where the Future Functionality attribute is employed for features planned for a previous deployment but were withheld.

Any text with no attribute is common to all versions when they are generated. However, when Fiat is selected in Show/Hide Conditional Text dialog box, any text or topic with a Chrysler attribute is omitted.

Generally, you can add an attribute to a word, phrase, etc. However, tables are more complicated. There is no way to omit an entire row of a table, which results in a blank row. The workaround for this is a separate table for each version/attribute. A good example of this is the Routines section of SONUS topic r_Validation_Results_Configuration.xml.

Entire Topics

To apply the Conditional Text attribute to an entire topic:
  1. Go to Resource Manager > File > Switch to XML View of Map. The map displays in View.
  2. Locate and select the topic title including the <topicref> tags.
  3. Go to Reuse > Apply/Remove Conditions and select the desired attribute.
  4. Click Apply, then OK to close the dialog box.
  5. Ctrl+S to save and Ctrl+W to close the map file. Unfortunately, there is no way to move the map from View to Resource Manager.
