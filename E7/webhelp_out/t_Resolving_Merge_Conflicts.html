Resolving Merge Conflicts

How to resolve files in conflict following a merge function.

Files which have been revised and checked in, then revised again display as having, or being in a state of conflict. Files which have been deleted from the working copy display as having a tree-conflict. The following procedure is recommended for resolving these issues:
Note: This procedure is based on the SmartSVN user interface, which is recommended. However, this concept is easily applied to the TortoiseSVN.
  1. In the Files container, select the file in conflict:
    • For conflict items, double-click a file name.
    • For tree-conflict items, select Mark Resolved from the context menu to remove the conflict.
    A dialog displays the Remote and Local (working copy) versions of the file in a side-by-side view.

  2. Assuming you are merging the branch to trunk, your objective is to move the Local Copy revisions to the Pristine Copy version. Referring to the illustration above, the green shaded items represent revisions to be merged. Click the corresponding icon for each Local Copy revision and leave the Pristine Copy icons untouched.
  3. Click Save and close the dialog box. The Files container refreshes to display a Modified status.
