Merge a Branch to the Trunk

Post release: merging revisions into the Trunk.

Most SVN operations may be performed with the SVN Client. However, issues may arise due to incompatible client/server versions, or other unknowns, where an alternative is required. See Merge With SmartSVN should you experience complications, such as multiple tree conflict warnings.

  1. Commit any revisions of the local branch folder.
  2. If necessary, Update local trunk folder. This is only required when more than users are updating files. If you are the only Writer, you can skip this step.
  3. Tag local branch folder to SVN repository with version ID.
    1. In the SVN browser, select Copy to from the context menu of the branch version folder, e.g. svn://red-repo.securityinspection.com/repos/HelpDocumentation/GODDARDHelp/branches/9.4.0. A dialog box displays with a text field to type the desired target directory. By default, SVN populates this text field with the directory of the folder you are copying.
    2. Revise the directory from .../branches/... to .../tags/... and click OK. The dialog box close.
    3. Verify the tag folder exists.
  4. In the local directory, select Merge from context menu of local trunk folder. The Merge wizard displays.
  5. Select Merge a range of revisions and click Next. The wizard advances to Merge revision range where you select the branch folder to merge into the local trunk folder.
  6. Set the following Merge revision range options accordingly:
    1. In URL to merge from, type or select (using either the drop-down menu or the browse button) the SVN branch directory.
    2. Enter revision version number(s) in Revision range to merge.
    3. Verify Working Copy directory is local trunk folder and click Next.
    The wizard advances to Merge options.
  7. Set the following Merge options options accordingly:
    • Merge options = Merge depth: Working copy
    • Ignore all whitespaces
    • All check boxes are clear.
  8. Optionally (recommended), click Test merge. A dialog box displays with projected results.
  9. Click Merge. It is common for files having multiple revisions to display with a conflict. Deleted files display with a tree-conflict. See Resolving Merge Conflicts for additional information. The Merge wizard closes and another dialog box displays files as they are merged. The OK button enables when the merge is complete.
  10. Click OK. The dialog box closes.
  11. Verify revisions exist in the local trunk folder and select SVN Commit from the context menu.
  12. If necessary, rename repository branch version folder, select Switch from the context menu of the local branch folder, and revise the folder name.
