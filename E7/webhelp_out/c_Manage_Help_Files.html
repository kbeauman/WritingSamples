Manage Help Files

Version Control and file backup.

E7 uses TortoiseSVN version control for managing files. TortoiseSVN Client is a free open-source program for version control and is available at http://tortoisesvn.net/.

Help File Folder Structure

E7 employs the industry standard of using trunks, branches, and tags to manage development cycles. The general practice for developers is to use a branch to check in revisions, which are then merged into the trunk. Tag folders are used to store released versions for reference and possible roll-backs. However, using this procedure for Help is a bit labor-intensive and offers few benefits. Therefore, it is recommended to Check files out to the trunk and reserve branch folders for situations where multiple versions are under development and it is necessary to keep these versions separate. See Merge a Branch to the Trunk for instructions on working with branch folders.
CAUTION:
Each folder level of the Working Copy contains a hidden folder "\.svn". This is used by TortoiseSVN for source control and should be considered off limits by novice users.

Following a release, the trunk folder is used to generate a tag version.
