SVN Post Deployment Procedure

Tag the Branch and merge to the Trunk.

Prerequisites:

All working copies must be checked in.

  1. Tag the Branch: Open the SVN Browser to svn://red-repo.securityinspection.com/repos/HelpDocumentation. User Name: kevin, Password: qxz934.
  2. Navigate to the desired folder, right-click and select Copy to from the context menu. A dialog box displays with the current file directory.
  3. Edit the file directory by replacing /branches/ with /tags/ and click OK. The dialog box closes.
  4. Explode the tags folder to verify.
  5. Merge to the Trunk: In your working copy, right-click the trunk folder and go to TortoiseSVN > Merge. A wizard displays Merge type.
  6. Select Merge a range of revisions and click Next. The wizard advances to Merge revision range.
  7. If necessary, edit the URL to merge from to the SVN branch folder. Alternatively, use the browse button (labeled with an eclipse) to navigate to the folder.
  8. Enter the Revision range to merge from separated by a hyphen (no spaces).
    1. Alternatively, click Show log to see a complete list of revisions for the branch folder. A second dialog box displays.
    2. Select both the earliest and latest log entries.
    3. Click OK. The dialog box closes to display the selected revisions.
    4. Edit the comma to a hyphen.
  9. Verify the Working Copy directory is ../trunk and click Next. The wizard advances to Merge options.
  10. Set the following options:
    1. Merge depth - Working copy.
    2. Ignore all whitespaces - Selected.
    3. All check boxes are clear.
  11. Click Merge. The wizard closes and another dialog box displays the progress. When the merge is complete OK is enabled.
  12. Click OK. The dialog box closes.
  13. Verify revisions exist in working copy trunk folder, right-click and select SVN Commit from the context menu.
