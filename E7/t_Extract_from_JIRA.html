Extract from JIRA

Release Notes are derived from the DPTCCB Project tickets of JIRA.

  1. In JIRA, go to the Issues tab and configure the following columns to display (Tools > Configure Columns ): T (Issue Type), Key, Summary, Release Notes Description.
  2. Search for Overview filters. There are filters for each release by product with a prefix of "Overview". To view Overview filters, go to Dashboards > Manage Filters > Search and type overview in the Search field. All Overview filters display for all products and all releases.
  3. Right-click each desired filter(s) and select Open Link in New Tab from the context menu. All tickets for the selected release display in each new tab.
  4. Go to Views > Excel (Current fields) to download the report. A dialog box displays to Open or Save the file.
  5. Select Save As and click OK. A second dialog box displays to select a location and rename the file. For one of the reports, it is recommended to rename the file "ReleaseNotesDDMMMYYY.xls", where DDMMMYYY is the Date, Month, and Year of the release. All other reports may retain the default name. This file serves as the final workbook for the release.
  6. When all reports for the upcoming release have been downloaded as .xls files, bring the ReleaseNotesDDMMMYYY.xls file to focus.
  7. Rename the tab to the product name and Save. Repeat this for each worksheet.
  8. Right-click the tab of any file other than the ReleaseNotesDDMMMYYY.xls workbook and select Move or Copy from the context menu. The Move or Copy dialog box displays.
  9. Select ReleaseNotesDDMMMYYY.xls for To book and the desired Before sheet option, then click OK. The dialog box closes to display the moved tab in the ReleaseNotesDDMMMYYY.xls workbook.
  10. Continue the Move function for all remaining product worksheets and Save the workbook.
  11. It is recommended to Custom Sort (Home > Editing > Sort & Filter > Custom Sort) by T (Issue Type) descending and Key ascending. For the most part, this orders each tab to display the New Feature items first with the Key order reflecting the sequence of when the issue was created. However, because Excel reads from left to right, Key values with fewer numeric characters are often misplaced and must be Cut and Inserted accordingly. Example: SONUS-124 may be listed after SONUS-1124 and need to be moved.
  12. Save the workbook and submit to Edmond, Ross, and Ranjana for review.

Approved workbooks are forwarded to Andrew Whitley and used to author the KB Publishing articles discussed in Publish Release Notes.
