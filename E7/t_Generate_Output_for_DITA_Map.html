Generate Output for DITA Map

Publish the Help HTML file set.

Currently, each product has its own Open Toolkit, which contains all images, style sheets, etc. that are specific to the product. These Toolkits are stored in the svn://red-repo.securityinspection.com/repos/HelpDocumentation/Support/Custom_OTs directory.

Checkout the Custom_OTs folder to your harddrive. You can put the folder anywhere, keeping in mind that you need to enter the entire file path in step 7 of the following procedure.

  1. In Resource Manager, go to File > Generate output for DITA Map. The Generate Output for DITA Map dialog box displays with the selected map in the title bar.
  2. If necessary, select WebHelp from the Deliverable type menu.
    1. For SONUS, click Show/Hide Conditional Text. The Show/Hide Conditional Text dialog box displays with the available attributes.
    2. Explode Product to clear/check the necessary output options, e.g., for Fiat check Fiat and clear Chrysler.
    3. Click OK to close the dialog box. See Conditional Text for additional information.
  3. Click Configure Output. The Configure Output Options dialog box displays.
  4. On the Deliverable Types tab, click Edit. The Edit Deliverable Type dialog box displays.
  5. On the General tab, enter Stylesheet settings. It is recommended to use Browse since the complete path is required.
    • Set the Specific CSS to the complete path of the webhelp_out\commonltr.css file of the product directory. Example: C:\...\GODDARDHelp\branch\webhelp_out\commonltr.css.
    • Set the Default Footer (FTR) to the complete path of the webhelp_out\chrysler_custom-ftr.html file of the product directory. Example: C:\...\GODDARDHelp\branch\webhelp_out\chrysler_custom-ftr.html.
    Note: There is a custom footer for Fiat to be used in SONUS if applicable. The SONUS directory is a little more complicated in that the webhelp_out folder contains Chrysler and Fiat subfolders. Any global revision requires both versions to be generated. See Conditional Text for additional information.
  6. Click OK to close the dialog box, and click the Advanced tab of the Configure Output Options dialog box.
  7. In Other output parameters (<name>=<value> pairs): enter the following: DITA_OT_BUILD_DIR = followed by the complete path of the corresponding Custom_OT directory, e.g., C:\...\Custom_OTs\e7_OT. This is a one-time requirement for generating a Help file. However, the directory must be revised accordingly for each product.
  8. Click OK to close the dialog box, and click Browse of the Generate Output for DITA Map dialog box. A standard Windows dialog box displays.
  9. Navigate to and select the desired ...\webhelp_out\*Help.html file, then click Save. A dialog box displays requesting confirmation to replace the existing file.
  10. Click OK to close the dialog box, and click OK of the Generate Output for DITA Map dialog box. A dialog box displays with a lime green indicator to show the system is generating.
    Tip: In the event of the indicator not showing, XMetaL has probably froze up (it happens fairly often). Open the Task Manager (Ctrl+Alt+Delete) to verify XMetaL is not responding. You must Shut Down and reboot (Restart does not work for this).
A dialog box displays when the Help file has finished compiling. Click Open file to view the results.
